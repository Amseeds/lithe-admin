## 添加菜单页面流程

本项目采用**动态路由 + 菜单配置一体化**的设计模式，添加新菜单页面只需两个步骤：

### 步骤一：创建页面组件

在 `src/views/` 目录下创建对应的 Vue 组件文件。

**目录结构规范：**

```
src/views/
├── your-page/           # 菜单对应的文件夹
│   └── index.vue        # 页面组件
```

**组件模板示例：**

```
<script setup lang="ts">
defineOptions({
  name: 'YourPage', // 组件名称，用于 keep-alive
})
</script>

<template>
  <div class="p-4">
    <h1>页面标题</h1>
    <!-- 页面内容 -->
  </div>
</template>
```

### 步骤二：配置菜单项

在 `src/api/user.ts` 的 `menu` 数组中添加菜单配置：

```typescript
{
  path: 'your-page',              // 路由路径（子路由不需要前导斜杠）
  name: 'yourPage',               // 路由名称（唯一标识）
  label: '页面名称',               // 菜单显示文本
  icon: 'iconify ph--star',       // 菜单图标
  component: 'your-page/index',   // 组件路径（相对于 @/views/，不含 .vue）
  meta: {
    title: '页面标题',             // 页面标题
    componentName: 'YourPage',    // 组件名（用于 keep-alive）
    showTab: true,                // 是否显示标签页
    pinned: false,                // 是否固定标签页
  },
},
```

## 菜单配置字段说明

### 基础字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `path` | string | 是 | 路由路径 |
| `name` | string | 是 | 路由名称，必须唯一 |
| `label` | string | 是 | 菜单显示文本 |
| `icon` | string | 否 | 图标类名 |
| `component` | string | 是 | 组件路径 |


### Meta 元信息

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `title` | string | 页面标题 |
| `componentName` | string | 组件名（keep-alive 缓存键） |
| `showTab` | boolean | 是否显示标签页 |
| `pinned` | boolean | 是否固定标签页 |
| `enableMultiTab` | boolean | 是否允许多标签实例 |
| `keepAlive` | boolean | 是否缓存组件状态 |


## 菜单类型示例

### 1. 单级菜单

```typescript
{
  path: 'dashboard',
  name: 'dashboard',
  label: '仪表板',
  icon: 'iconify ph--chart-line',
  component: 'dashboard/index',
},
```

### 2. 多级菜单（带子菜单）

```typescript
{
  path: 'data-show',
  name: 'dataShow',
  label: '数据展示',
  icon: 'iconify ph--database',
  redirect: 'data-show/data-table',
  children: [
    {
      path: 'data-table',
      name: 'dataTable',
      label: '数据表格',
      icon: 'iconify ph--table',
      component: 'data-show/data-table/index',
    },
    {
      path: 'data-form',
      name: 'dataForm',
      label: '数据表单',
      icon: 'iconify ph--article',
      component: 'data-show/data-form/index',
    },
  ],
},
```

### 3. 带分组的菜单

```typescript
{
  path: 'settings',
  name: 'settings',
  label: '系统设置',
  children: [
    {
      type: 'group',
      key: 'basic-group',
      label: '基础设置',
      children: [
        { path: 'profile', name: 'profile', label: '个人资料', component: 'settings/profile/index' },
      ],
    },
    { type: 'divider', key: 'divider-1' }, // 分割线
    {
      type: 'group', 
      key: 'advanced-group',
      label: '高级设置',
      children: [
        { path: 'security', name: 'security', label: '安全设置', component: 'settings/security/index' },
      ],
    },
  ],
},
```

### 4. 动态路由（带参数）

```typescript
{
  path: 'detail/:id',
  name: 'detail',
  label: '详情页',
  component: 'detail/index',
  meta: {
    showTab: true,
    enableMultiTab: true, // 允许打开多个标签
    renderTabTitle: ({ id }) => `详情-${id}`, // 动态标签标题
  },
},
```

## 图标使用

项目使用 [Iconify](https://icon-sets.iconify.design/) 图标库，常用格式：

- `iconify ph--icon-name` - Phosphor 图标
- `icon-[collection--icon-name]` - 其他图标集

推荐图标集：

- [Phosphor Icons](https://phosphoricons.com/) - `ph--xxx`
- [Material Design Icons](https://materialdesignicons.com/) - `mdi--xxx`

## 完整示例

假设要添加一个「用户管理」页面：

**1. 创建组件** `src/views/user-management/index.vue`：

```
<script setup lang="ts">
import { ScrollContainer } from '@/components'

defineOptions({
  name: 'UserManagement',
})
</script>

<template>
  <ScrollContainer>
    <h1 class="text-xl font-semibold">用户管理</h1>
    <!-- 页面内容 -->
  </ScrollContainer>
</template>
```

**2. 添加菜单配置** 在 `src/api/user.ts` 的 menu 数组中：

```typescript
{
  path: 'user-management',
  name: 'userManagement',
  label: '用户管理',
  icon: 'iconify ph--users',
  component: 'user-management/index',
  meta: {
    title: '用户管理',
    componentName: 'UserManagement',
    showTab: true,
  },
},
```

## 注意事项

1. **组件路径格式**：使用 `folder/index` 格式，不需要 `.vue` 后缀
2. **路由名称唯一**：`name` 字段在整个应用中必须唯一
3. **子路由路径**：子菜单的 `path` 不需要前导斜杠
4. **图标一致性**：保持图标风格统一
5. **生产环境**：实际菜单数据由后端 API 返回，前端配置仅用于开发测试
