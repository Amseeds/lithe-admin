## 用户需求

将现有"控糖目标"模块从单层分类菜单重构为三级展开菜单结构，基于中国糖尿病防治指南2024版设计专业的分类体系。

## 功能内容

1. **模块总览**（保留现有"全部患者"功能）

- 患者分类占比饼图
- HbA1c分布饼图
- 通用控糖指标卡片
- 全院患者列表

2. **三级菜单结构**

- 一级：糖尿病类型分类（7大类）
- 二级：具体分型（如T2DM下分6个子类型）
- 三级：子分类（如老年T2DM按功能状态分层）

3. **右侧页面主体**（保持原有样式）

- 分类信息条（定义说明）
- 专属控糖目标卡片
- 统计卡片区
- 患者列表

4. **Mock数据补充**

- 21个叶子节点的分类定义
- 每个节点的控糖目标参数
- 患者人数统计（总计约200人）
- 患者列表数据

## 技术栈

- 框架: Vue 3 + TypeScript
- 语法: Composition API with `<script setup>`
- UI组件库: Naive UI (NMenu 支持多级展开)
- 图表: ECharts
- 样式: Tailwind CSS

## 实现方案

### 数据结构设计

```typescript
// 三级菜单节点类型
interface MenuCategory {
  key: string
  name: string
  children?: MenuCategory[]
}

// 叶子节点数据
interface CategoryData {
  key: string
  name: string
  parentPath: string[]  // 面包屑路径
  definition: string
  targets: Array<{ name: string; value: string }>
  patientCount: number
  statistics: {
    total: number
    controlled: number
    uncontrolled: number
    hypoglycemiaRisk: number
    rate: string
  }
}
```

### NMenu 多级菜单实现

Naive UI 的 MenuOption 支持 children 属性，可直接实现树形展开：

```typescript
const menuOptions: MenuOption[] = [
  { label: '模块总览', key: 'overview' },
  {
    label: '2型糖尿病（T2DM）',
    key: 't2dm',
    children: [
      { label: '普通成人T2DM', key: 't2dm-adult' },
      {
        label: '老年T2DM分层',
        key: 't2dm-elderly',
        children: [
          { label: '老年健康型', key: 't2dm-elderly-healthy' },
          { label: '老年功能受损型', key: 't2dm-elderly-impaired' },
          { label: '老年衰弱型', key: 't2dm-elderly-frail' },
        ]
      },
      // ...
    ]
  }
]
```

### 关键实现要点

1. **叶子节点识别**：只有叶子节点才显示分类详情页，非叶子节点仅作为导航分组
2. **状态同步**：使用 `expandedKeys` 控制菜单展开状态
3. **数据映射**：使用扁平 Map 结构快速查找叶子节点数据
